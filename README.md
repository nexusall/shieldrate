# ShieldRate
Defending distributed APIs against malicious traffic spikes.

## 分布式接口防刷（限流）系统设计与实现

**题目背景：**
在实际的业务场景中，为了防止恶意刷接口、爬虫爬取或突发流量打垮服务，我们需要对 API 接口进行访问频率限制（即“防刷”）。请你设计并实现一个带有防刷功能的 HTTP 接口服务。

**核心需求：**
1. **基础接口服务**
   - 实现一个简单的 HTTP 服务，包含 2-3 个业务接口（例如：`/api/user/info`、`/api/order/create` 等）。
   - 接口只需返回简单的 JSON 数据即可，无需实现复杂的业务逻辑。

2. **分布式防刷组件**
   - **防刷策略**：实现一个基于 **IP 地址** 的限流工具/组件。
   - **可配置性**：防刷规则需要灵活可配。例如，可以设置“每个 IP 在 N 秒内最多访问 M 次”。
   - **易集成性**：该防刷工具需要能够方便地配置/挂载到任意的 HTTP 接口上。
   - **分布式存储**：服务要求支持分布式部署。可以使用外部存储（如 Redis、Memcached 或数据库）来保存访问计数和状态。

3. **技术栈要求**
   - 编程语言、Web 框架、外部存储中间件等可自行选用。

**交付物要求：**
- 完整的项目源代码。
- 最好有一个简短的视频，说明整体的技术方案及设计思路，要求覆盖开发过程中遇到的问题，并说明如何解决。演示如何启动服务、如何配置限流规则。如何验证限流效果。
